与 v1 的对比口径警告:v1 报告(`docs/baseline_comparison.md`)在 3,000 个岗位的小池上排序,数字(如 Rule 0.121)与本报告**不可比**——池越小指标越高。本报告在 91,479 个岗位的全库上排序,这是难得多但更真实的设定。

## 2. 主对比表(test,全库排序,4,095 用户)

可训练模型为 3 seeds(42/43/44)均值 ± 标准差;确定性模型单次。

| 模型 | NDCG@10 | Recall@10 | Recall@50 | NDCG@50 | MRR |
|---|---:|---:|---:|---:|---:|
| Popularity | 0.0016 | 0.0039 | 0.0164 | 0.0043 | 0.0019 |
| SBERT 零样本(bge-small-zh-v1.5) | 0.0030 | 0.0051 | 0.0149 | 0.0050 | 0.0032 |
| BM25(jieba) | 0.0047 | 0.0093 | 0.0281 | 0.0087 | 0.0047 |
| Rule v2 | 0.0147 | 0.0305 | 0.1038 | 0.0303 | 0.0151 |
| LightGCN(d=128, 3 层) | 0.0233 ± 0.0011 | 0.0446 ± 0.0017 | 0.1027 ± 0.0008 | 0.0360 ± 0.0009 | 0.0203 ± 0.0009 |
| **SPC-HGT v2(最终)** | **0.0266 ± 0.0005** | **0.0513 ± 0.0006** | **0.1231 ± 0.0024** | **0.0424 ± 0.0002** | **0.0245 ± 0.0005** |

- 最强 baseline:**LightGCN**(NDCG@10 = 0.0233)。
- 主模型相对最强 baseline:**NDCG@10 +14.2%,Recall@10 +15.0%,Recall@50 +19.9%**。
- 显著性:seed 42 配对 t 检验(按用户 NDCG@10,n=4,095):t = 2.364,**p = 0.018**。
- 结论:**主模型在全部指标上超过全部 baseline,判定有效**(docs/13 §2 规则)。

观察:
- Rule v2 借助干净技能词表已是不弱的非学习 baseline(NDCG@10 0.0147,v1 的 BM25/semantic_hash 在等价口径下接近随机)。
- 零样本 SBERT 较弱:用户画像文本(技能标签拼接)与 JD 文本的分布差异大,纯语义不足以建模"会投递什么"。
- 协同信号(LightGCN)是这一数据集上最强的单一信号;SPC-HGT v2 的价值在于把协同、文本、结构化、技能路径融合后仍能稳定增益。

## 3. 最终模型构成与消融(基准 = 最终模型,seed 42,NDCG@10 = 0.0273)

| 变体 | NDCG@10 | Δ | 结论 |
|---|---:|---:|---|
| SPC-HGT v2(最终) | 0.0273 | — | — |
| − 行为分级对比损失(InfoNCE) | 0.0183 | −33.0% | **贡献最大**:对齐 z_c/z_j 空间直接决定召回阶段质量 |
| − 技能路径注意力 | 0.0249 | −8.8% | 正贡献,且是解释输出的来源 |
| − ID 嵌入(纯内容) | 0.0252 | −7.7% | 协同记忆有效 |
| − SBERT 文本特征(随机初始化) | 0.0260 | −4.8% | 正贡献,主要帮助冷门岗位 |
| + 语义难负样本课程 | 0.0239 | −12.5% | **负贡献,已从最终模型移除**(见 §4) |

所有保留组件均有正贡献,满足 docs/12 §5 的"组件必须由消融证明"规则。

## 4. 负结果:语义难负样本课程为何被移除

设计初衷(docs/12 §4)是用"SBERT 最相似但无行为"的岗位作难负。消融显示它使 NDCG@10 从 0.0273 降到 0.0239。归因:本数据集的行为是**曝光受限**的——用户没投递某岗位大概率是因为没看到,而不是不匹配。把"语义最像的未交互岗位"当负样本,惩罚的恰是大量伪负样本(本应是好推荐),模型被迫把"像用户正例的岗位"往后排,直接伤害排序目标。这与 ConFit 等工作中难负有效的前提(全曝光或近全曝光的标注)不同。

保留代码与开关(`use_hard_neg`),供拿到曝光数据(impression log)后重新验证。

## 5. 最终模型配置

```yaml
# configs/v2_spc_hgt.yaml(最终)
dim: 128, layers: 2, lr: 1e-3, batch: 2048, epochs: 60, patience: 8
neg_per_pos: 4(均匀负采样), lambda_bce: 0.2, lambda_nce: 0.1
tau: satisfied 0.07 / delivered 0.10
use_path: true, use_text: true, use_id: true, use_hard_neg: false
```

推理为两阶段:内积召回 top-500 → 路径注意力评分头重排(详见 docs/12 §1)。

## 6. 已知局限与下一步

1. 行序划分 ≠ 真实时间切分;有时间戳后必须重跑。
2. 候选池排除了零行为岗位(178K 个 JD 无任何行为,无法评估)。
3. browsed 信号目前只进对比损失弱正对,未充分利用;可尝试曝光建模(IPS 加权)修正 §4 的伪负问题。
4. 路径注意力的解释输出(matched_skills + α 权重)已在模型内可用,但推理 API(`inference/service.py`)尚未接入 v2,需要后续工程化。
5. 超参未做系统搜索(只在 valid 上挑了少量配置),NDCG 仍有上探空间。
